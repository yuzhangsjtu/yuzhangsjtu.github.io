Earlier, through a senior lab mate's recommendation, I got an Alibaba Cloud ECS instance for half a year. But since my domain name hasn't been filed, I probably can't use it. Besides, Alibaba Cloud will eventually start charging — so finding a stable repository for my blog became a must-have.

I tried out [Gitee](https://gitee.com/) briefly: it can fork GitHub repos seamlessly. But it's quite slow and unstable. After some searching, I learned about Vercel (formerly Zeit). Their servers are reportedly in Taiwan, so I ran a quick test — blazing fast, with minimum latency as low as 50ms.

Vercel has heaps of advantages:

- Seamlessly links GitHub repos. After deploying locally, Vercel syncs instantly.
- UIs are clean and user-friendly.
- It’s free.

So, my blog is temporarily housed on Vercel servers, while me GitHub Page is kept. Both the github.io site and the Vercel site point to [yuzhangnju.me](https://yuzhangnju.me/).
